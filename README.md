# GitHubTest voor AI Prompts en Agents

In deze omgeving houden we onze Prompts en Agents bij. 
Wanneer je een nieuwe Prompt of Agent maakt, leg hem dan eerst hier vast. 
Het bijhouden van de Prompts IN de Agents werkt nl niet zo goed.

## Toegang en Versie controle
Iedereen binnen het bedrijf kan aanpassing en opmekringen maken - GitHub houdt de versiecontrole bij.
Wanneer je klaar bent kan je de Prompt in een Agent zetten, of copy/pasten naar de de eigenaar van de Agent - deze kan hem dan aanpassen

De structuur is UNIX gebaseerd. In elke folder staat of een andere folder of een documentm. Maar NOOIT beide. Dit houdt het overzichtelijk. Alleen deze Readme file is de uitzondering.

## Welke Prompt voor welke processen
We hebben Prompts per proces (Managament, Finance, HR ICT en Business Developemen, communicatie/Marketing, Sales en logistiek
En per onderdeel zien we drie soorten inzichting: strategisch, tactisch en Operationeel.
Dit komt tot uiting in de Prompts.
